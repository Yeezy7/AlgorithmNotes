``` c++
int bin_search(int *a, int n, int x){   //在数组a中找数字x，返回位置
    int left = 0, right = n;
    while (left < right) {
        int mid = left+(right-left)/2;
        if (a[mid] >= x)  right = mid;
        else    left = mid + 1;
        cout << a[mid]<<"   ";   //调试：打印出猜数的过程
    }
   return left;
```

## 二维数组中的查找

来源：[二维数组中的查找]https://www.nowcoder.com/practice/abc3fe2ce8e146608e868a70efebf62e?tpId=295&tags=&title=&difficulty=0&judgeStatus=0&rp=0&sourceUrl=%2Fexam%2Foj%3FquestionJobId%3D10%26subTabName%3Donline_coding_page
### 描述

在一个二维数组array中（每个一维数组的长度相同），每一行都按照从左到右递增的顺序排序，每一列都按照从上到下递增的顺序排序。请完成一个函数，输入这样的一个二维数组和一个整数，判断数组中是否含有该整数。

[[1,2,8,9],  
[2,4,9,12],  
[4,7,10,13],  
[6,8,11,15]]

给定 target = 7，返回 true。
给定 target = 3，返回 false。

数据范围：矩阵的长宽满足 0≤n,m≤5000≤n,m≤500 ， 矩阵中的值满足 −109≤val≤109−109≤val≤109  
进阶：空间复杂度 O(1)O(1) ，时间复杂度 O(n+m)O(n+m)  


### 💡 核心思路：线性查找

最直接的想法是暴力遍历，但时间复杂度是 O(n*m)，显然没有利用“有序”这个条件。

最优解法是从数组的**右上角**（或**左下角**）开始查找。这个位置很特殊，它**是所在行的最大值，同时是所在列的最小值**。

- 如果 `target` **大于**当前值，说明 `target` 不可能在这一行（因为当前值已经是本行最大），所以**向下移动一行**。
- 如果 `target` **小于**当前值，说明 `target` 不可能在这一列（因为当前值已经是本列最小），所以**向左移动一列**。
- 如果相等，直接返回 `true`。

这样每次都能排除一行或一列，时间复杂度仅为 **O(n+m)**[](https://www.nowcoder.com/practice/abc3fe2ce8e146608e868a70efebf62e?tpId=295&tags=&title=&difficulty=0&judgeStatus=0&rp=0&sourceUrl=%2Fexam%2Foj%3FquestionJobId%3D10%26subTabName%3Donline_coding_page)。

```c++
#include <vector>
using namespace std;

class Solution {
public:
    bool Find(int target, vector<vector<int> > array) {
        // 处理空数组的情况
        if (array.empty() || array[0].empty()) {
            return false;
        }

        int rows = array.size();
        int cols = array[0].size();

        // 从右上角开始 (第0行，最后一列)
        int row = 0;
        int col = cols - 1;

        // 循环条件：不超出数组边界
        while (row < rows && col >= 0) {
            int current = array[row][col];
            
            if (current == target) {
                return true; // 找到了
            } else if (current < target) {
                // 当前值小于目标，排除本行，下移
                row++;
            } else {
                // 当前值大于目标，排除本列，左移
                col--;
            }
        }

        // 循环结束，说明没找到
        return false;
    }
};
```

## 寻找峰值
来源：[]https://www.nowcoder.com/practice/fcf87540c4f347bcb4cf720b5b350c76?tpId=295&tags=&title=&difficulty=0&judgeStatus=0&rp=0&sourceUrl=%2Fexam%2Foj%3FquestionJobId%3D10%26subTabName%3Donline_coding_page


### 描述

给定一个长度为n的数组nums，请你找到峰值并返回其索引。数组可能包含多个峰值，在这种情况下，返回任何一个所在位置即可。

1.峰值元素是指其值严格大于左右相邻值的元素。严格大于即不能有等于
2.假设 nums[-1] = nums[n] = −∞−∞
3.对于所有有效的 i 都有 nums[i] != nums[i + 1]
4.你可以使用O(logN)的时间复杂度实现此问题吗？

数据范围：

1≤nums.length≤2×105 1≤nums.length≤2×105   

−231<=nums[i]<=231−1−231<=nums[i]<=231−1  

如输入[2,4,1,2,7,8,4]时，会形成两个山峰，一个是索引为1，峰值为4的山峰，另一个是索引为5，峰值为8的山峰

方式一：暴力搜索

```c++
class Solution {
public:
    int findPeakElement(vector<int>& nums) {
        // write code here
        // 数组长度只有1个
        int n = nums.size();
        if(n == 1){
            return 0;
        }
        // 先判断两边
        if(nums[0] > nums[1]){
            return 0;
        }
        if(nums[n-1] > nums[n-2]){
            return n-1;
        }

        for(int i=1;i<n-1;++i){
            if(nums[i] > nums[i-1] && nums[i] > nums[i+1]){
                return i;
            }
        }
        return -1;
    }
};
```

方式二：二分查找
因为二分查找的本质是`二段性`，二分查找的过程本质是对`可行区间的压缩`。只要满足二段性的问题都可以用二分查找解决。在这里二段性的体现是峰值的左边单调增，右边单调减。你可能会反驳给我们的数值不只有一个峰值，但是只要我们控制好条件，一定可以把范围压缩到只有一个峰值的情况，来看看该怎么处理：
- nums[mid] < nums[mid + 1]说明在“上坡”，则可以使left = mid + 1（因为mid肯定不是峰值），向“峰”处压缩
- nums[mid] > nums[mid + 1]说明在“下坡”，则应该使right = mid（mid可能是峰值），往“峰”处压缩
虽然开始left和right之间可能有多个峰值，但是随着left和right不断逼近，最后两者之间一定会压缩到一个峰值上，因为两者都是向“峰”不断靠近的，但是不会超过`最终`的“峰”
```c++
int findPeakElement(vector<int>& nums) {
    // write code here
    // 数组长度只有1个
    int left = 0, right = nums.size() - 1;
    while(left < right){
        int mid = (left + right) / 2;
        if(nums[mid] < nums[mid+1]){
            left = mid + 1;
        }
        else{
            right = mid;
        }
    }
    return left;
}
```
