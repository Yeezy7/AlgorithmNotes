好的，我们来详细讲解一下 KMP（Knuth-Morris-Pratt）算法。这是一个经典的字符串匹配算法，用于在一个主串（文本）中查找一个模式串（Pattern）出现的位置。

### 1. 朴素匹配的问题在哪？

假设我们有：
- 主串 `S = "ABCABCABD"`
- 模式串 `P = "ABCABD"`

如果使用暴力匹配（双循环），当发生失配时，我们会把 `P` **向右移动 1 位**，然后从头开始比较。

```
S: A B C A B C A B D
P: A B C A B D
      ↑ 匹配到这里失配

朴素做法：
S: A B C A B C A B D
P:   A B C A B D  （P右移1位，j重置为0）
```

但仔细看失配的位置：我们已经成功匹配了 `"ABCAB"` 这 5 个字符。在这个前缀里，最前面的 `"AB"` 和最后面的 `"AB"` 是相同的。既然后面的 `"AB"` 刚刚匹配成功，那么我们把 `P` 的开头 `"AB"` 直接对准主串刚匹配成功的后面那个 `"AB"`，不是更好吗？这样就不用比较已经知道相等的部分了。

KMP 的核心思想就是：**利用已经部分匹配的信息，保持主串指针 `i` 不回溯，只移动模式串指针 `j`，从而把时间复杂度降到 O(n+m)**。

### 2. 前缀、后缀与 Next 数组

为了实现“智能跳跃”，我们需要预处理模式串 `P`，得到一个 `next` 数组（有时也叫 `lps` —— 最长相同前缀后缀长度）。

**定义**：对于模式串 `P` 的前缀子串 `P[0..k]`，其 **最长相同前缀后缀长度**（Longest Proper Prefix which is also Suffix，简称 LPS）是指：该子串的 **真前缀** 和 **真后缀** 的最大匹配长度。（注意：不能是整个子串本身）。

**例子**：模式串 `P = "ABCABD"`

| 子串 | 最长相同前缀后缀 | 长度 |
| :--- | :--- | :--- |
| "A" | 无 | 0 |
| "AB" | 无 | 0 |
| "ABC" | 无 | 0 |
| "ABCA" | "A" | 1 |
| "ABCAB" | "AB" | 2 |
| "ABCABD" | 无 | 0 |

这就是 `lps` 数组：`[0, 0, 0, 1, 2, 0]`。

在 KMP 具体实现时，`next` 数组通常有两种常见变体：
1. 直接用 `lps` 值。
2. 将 `lps` 值右移一位，并在开头补 `-1`，即 `next[j]` 表示当 `P[j]` 失配时，`j` 应该跳转到的下标（或者下标减 1，取决于代码写法）。

为便于理解，这里我们先按照 **`next[j]` 表示 `P[0..j-1]` 的最长相同前缀后缀长度** 来讲解（即 `next[0] = -1`），这也是比较经典的实现方式。

### 3. 如何计算 Next 数组（递推）

计算 `next` 数组本身也是一个“模式串自我匹配”的过程。

```c++
vector<int> getNext(const string& p) {
    int m = p.size();
    vector<int> nxt(m, -1);
    int i = 0, j = -1;
    while (i < m - 1) {
        if (j == -1 || p[i] == p[j]) {
            ++i; ++j;
            nxt[i] = j;
        } else {
            j = nxt[j];
        }
    }
    return nxt;
}
```

**手动模拟计算 `"ABCABD"` 的 next：**

- 初始：`i=0, j=-1, next[0]=-1`
- `j==-1` → `i=1, j=0` → `next[1]=0`
- `P[1]='B' != P[0]='A'` → `j=next[0]=-1`
- `j==-1` → `i=2, j=0` → `next[2]=0`
- `P[2]='C' != P[0]='A'` → `j=-1` → `i=3, j=0` → `next[3]=0`
- `P[3]='A' == P[0]='A'` → `i=4, j=1` → `next[4]=1`
- `P[4]='B' == P[1]='B'` → `i=5, j=2` → `next[5]=2`

得到 `next = [-1, 0, 0, 0, 1, 2]`。

**`next` 数组的含义**：
- 当 `P[5]`（即 `'D'`）失配时，`j` 跳转到 `next[5] = 2`，也就是 `P[2]`（即 `'C'`）继续比较。因为前 5 个字符 `"ABCAB"` 的最长相同前后缀是 `"AB"`，长度为 2，我们已经匹配了后缀的 `"AB"`，自然可以把前缀的 `"AB"` 对准位置。

### 4. KMP 匹配过程

有了 `next` 数组，匹配逻辑就很简单了：

```python
def kmp_search(s, p):
    n, m = len(s), len(p)
    nxt = get_next(p)
    i = 0  # 主串指针
    j = 0  # 模式串指针
    
    while i < n and j < m:
        if j == -1 or s[i] == p[j]:
            # 当前字符匹配，或 j 已回溯到 -1（需重新开始）
            i += 1
            j += 1
        else:
            # 失配，i 不动，j 跳转到 next[j]
            j = nxt[j]
    
    if j == m:
        # 匹配成功，返回起始位置
        return i - j
    else:
        return -1
```

**演示匹配 `S = "ABCABCABD"` 和 `P = "ABCABD"`**：

1. `i=0, j=0` 开始匹配，连续匹配 5 个字符：
   ```
   S: A B C A B C A B D
   P: A B C A B D
   ```
   当 `i=5, j=5` 时，`S[5]='C'`，`P[5]='D'`，失配。

2. 查询 `next[5] = 2`，`j` 跳转到 2，`i` 保持 5：
   ```
   S: A B C A B C A B D
   P:       A B C A B D
            ↑ j=2 这里
   ```
   现在比较 `S[5]='C'` 和 `P[2]='C'`，相等！`i=6, j=3`。

3. 继续比较，`S[6]='A' == P[3]='A'`，`S[7]='B' == P[4]='B'`，`S[8]='D' == P[5]='D'`，匹配完成。返回 `i - j = 9 - 6 = 3`，即模式串出现在主串下标 3 的位置。

### 5. 优化：NextVal 数组

在计算 `next` 时，如果 `P[i] == P[next[i]]`，那么失配跳转后比较的字符还是一样，必然继续失配。我们可以直接把 `next[i]` 改成 `next[next[i]]`，这就是 `nextval` 优化。

```python
def get_nextval(p):
    m = len(p)
    nxt = [-1] * m
    i, j = 0, -1
    while i < m - 1:
        if j == -1 or p[i] == p[j]:
            i += 1
            j += 1
            if p[i] != p[j]:
                nxt[i] = j
            else:
                nxt[i] = nxt[j]  # 如果字符相同，直接继承
        else:
            j = nxt[j]
    return nxt
```

### 6. 复杂度分析

- **时间复杂度**：计算 `next` 数组 O(m)，匹配过程 O(n)，总时间复杂度 **O(n+m)**。
- **空间复杂度**：O(m)，用于存储 `next` 数组。

### 7. 总结

KMP 算法的精髓在于 **next 数组**，它记录了模式串自身的重复结构信息。当匹配失败时，我们不是像暴力算法那样回溯主串指针，而是根据已经匹配的部分，将模式串移动到一个合理的位置继续比较。这使得 KMP 在最坏情况下也能保持线性效率，是字符串匹配领域的一个里程碑式算法。
