## 原地归并两个升序链表

迭代法 + 虚拟头节点：空间复杂度为 O(1)

```c++
/**
 * struct ListNode {
 *     int val;
 *     struct ListNode *next;
 *     ListNode(int x) : val(x), next(nullptr) {}
 * };
 */
class Solution {
public:
    ListNode* Merge(ListNode* pHead1, ListNode* pHead2) {
        // 1. 创建虚拟头节点（辅助节点，不存实际数据）
        ListNode* dummy = new ListNode(0);
        ListNode* cur = dummy;  // cur 始终指向新链表的尾节点

        // 2. 同时遍历两个链表，比较节点值
        while (pHead1 != nullptr && pHead2 != nullptr) {
            if (pHead1->val <= pHead2->val) {
                cur->next = pHead1;   // 将 pHead1 接到新链表尾部
                pHead1 = pHead1->next;
            } else {
                cur->next = pHead2;
                pHead2 = pHead2->next;
            }
            cur = cur->next;          // 移动尾指针
        }

        // 3. 将剩余未合并的部分直接接上
        cur->next = (pHead1 == nullptr) ? pHead2 : pHead1;

        // 4. 返回合并后的链表头（跳过虚拟节点）
        return dummy->next;
    }
};
```

## 链表中环的入口节点

来源：[牛客 BM7 ]https://www.nowcoder.com/practice/253d2c59ec3e4bc68da16833f79a38e4?tpId=295&tags=&title=&difficulty=0&judgeStatus=0&rp=0&sourceUrl=%2Fexam%2Foj%3FquestionJobId%3D10%26subTabName%3Donline_coding_page

下面我将为你详细拆解这个算法的思路、证明以及C++实现。

### 1. 算法核心思路（分两步走）

1.  **判断是否有环**：使用快慢指针，快指针 `fast` 每次走两步，慢指针 `slow` 每次走一步。如果链表有环，它们最终一定会在环内的某个节点相遇；如果无环，快指针会先到达链表末尾。
2.  **找到环的入口**：当 `fast` 和 `slow` 第一次相遇后，将 `fast` (或 `slow`) 重新指向链表头节点。然后，让这两个指针**每次都只走一步**，它们再次相遇的节点就是环的入口节点。

---

### 2. 数学证明

这个算法的巧妙之处在于其背后的数学原理。我们定义三个关键距离：

-   **`a`**：链表头节点到环入口节点的距离。
-   **`b`**：环入口节点到快慢指针第一次相遇节点的距离。
-   **`c`**：第一次相遇节点到环入口节点的距离（沿着环的方向）。

因此，环的周长 `L = b + c`。

当快慢指针第一次相遇时：

-   慢指针 `slow` 走过的距离为：`a + b`
-   快指针 `fast` 走过的距离为：`a + b + n*(b+c)`，其中 `n` 是快指针在相遇前绕环的圈数（`n >= 1`）。

因为快指针速度是慢指针的2倍，所以有：
`2 * (a + b) = a + b + n*(b + c)`
化简后得到：
`a = (n-1)*(b+c) + c` 
即：
`a = (n-1)*L + c` 

**这个等式的含义是**：从链表头节点走到环入口的距离 `a`，等于从相遇点出发，绕环走 `(n-1)` 圈，再走 `c` 的距离。

所以，当我们将一个指针放回链表头，另一个留在相遇点，然后让它们以相同的速度（每次一步）前进时：
-   从链表头出发的指针走 `a` 步到达环入口。
-   从相遇点出发的指针走 `(n-1)*L + c` 步也会到达环入口。

因此，它们必定会在环的入口节点再次相遇。

---

### 3. C++ 代码实现

根据上述思路，代码实现非常清晰：

```cpp
/*
struct ListNode {
    int val;
    struct ListNode *next;
    ListNode(int x) :
        val(x), next(NULL) {
    }
};
*/
class Solution {
public:
    ListNode* EntryNodeOfLoop(ListNode* pHead) {
        // 1. 定义快慢指针
        ListNode* fast = pHead;
        ListNode* slow = pHead;

        // 2. 第一步：判断是否有环，并找到相遇点
        while (fast != nullptr && fast->next != nullptr) {
            slow = slow->next;          // 慢指针走一步
            fast = fast->next->next;    // 快指针走两步

            // 找到相遇点
            if (slow == fast) {
                // 3. 第二步：找到环的入口
                // 将快指针（或慢指针）重新指向链表头
                fast = pHead;
                // 两个指针每次都走一步，直到相遇
                while (fast != slow) {
                    fast = fast->next;
                    slow = slow->next;
                }
                // 相遇的节点就是环的入口
                return fast;
            }
        }

        // 链表无环，返回 nullptr
        return nullptr;
    }
};
```

### 4. 复杂度分析

-   **时间复杂度**：`O(n)`。其中 `n` 为链表节点数。以慢指针为基准，整个流程最多遍历链表两次，因此是线性时间复杂度。
-   **空间复杂度**：`O(1)`。只使用了两个指针变量，没有使用额外的数据结构。

### 💎 总结

这个算法的精妙之处在于，它将一个看似复杂的“找入口”问题，通过数学公式 `a = (n-1)*L + c` 转换为了一个简单的“两人从不同起点、同速前进，必定在终点相遇”的问题。掌握这个证明过程，能帮助你更深刻地理解快慢指针的应用场景。
