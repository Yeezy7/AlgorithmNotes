## 引入

前缀和与差分是算法竞赛中常用的技巧，前者用于快速求区间和，后者用于高效进行区间修改．

> 规定
> 为方便讨论，本文默认数组 {𝑎𝑖}![](data:image/gif;base64,R0lGODlhAQABAIAAAAAAAP///yH5BAEAAAAALAAAAAABAAEAAAIBRAA7 "\{a_i\}") 下标从 1![](data:image/gif;base64,R0lGODlhAQABAIAAAAAAAP///yH5BAEAAAAALAAAAAABAAEAAAIBRAA7 "1") 开始，并补充定义 𝑎0 =0![](data:image/gif;base64,R0lGODlhAQABAIAAAAAAAP///yH5BAEAAAAALAAAAAABAAEAAAIBRAA7 "a_0 = 0")．

# 前缀和

前缀和可以简单理解为「数列的前 𝑛![](data:image/gif;base64,R0lGODlhAQABAIAAAAAAAP///yH5BAEAAAAALAAAAAABAAEAAAIBRAA7 "n") 项的和」，是一种重要的预处理方式．

### 一维前缀和

对于长度为 𝑛![](data:image/gif;base64,R0lGODlhAQABAIAAAAAAAP///yH5BAEAAAAALAAAAAABAAEAAAIBRAA7 "n") 的序列 {𝑎𝑖}![](data:image/gif;base64,R0lGODlhAQABAIAAAAAAAP///yH5BAEAAAAALAAAAAABAAEAAAIBRAA7 "\{a_i\}")，如果要多次查询区间 [𝑙,𝑟]中序列数字的和，就可以考虑使用前缀和．序列的前缀和就是

𝑆𝑖=𝑖∑𝑗=1𝑎𝑗.

它可以通过递推关系式

𝑆0=0, 𝑆𝑖=𝑆𝑖−1+𝑎𝑖
逐项计算得到．要询问区间 [𝑙,𝑟]内的序列的和，只需要计算差值

𝑆([𝑙,𝑟])=𝑆𝑟−𝑆𝑙−1.
就这样，通过 𝑂(𝑛)时间预处理，能够将单次查询区间和的复杂度降低到 𝑂(1)．

参考实现

```c++
int n;                // Array size.
std::vector<int> a;   // Array. (indexed from 1)
std::vector<int> ps;  // Prefix sum array.

// Calculate prefix sum.
void prefix_sum() {
  ps = a;
  // Or simply:
  // std::partial_sum(a.begin(), a.end(), ps.begin());
  for (int i = 1; i <= n; ++i) {
    ps[i] += ps[i - 1];
  }
}

// Query sum of elements in [l, r].
int query(int l, int r) { return ps[r] - ps[l - 1]; }
```

C++ 标准库中实现了前缀和函数 [`std::partial_sum`](https://zh.cppreference.com/w/cpp/algorithm/partial_sum)，定义于头文件 `<numeric>` 中．从 C++17 开始，标准库还提供了一个功能相同的前缀和函数 [`std::inclusive_scan`](https://zh.cppreference.com/w/cpp/algorithm/inclusive_scan)，同样定义于头文件 `<numeric>` 中．

### 二维/多维前缀和

将一维前缀和拓展到多维的情形，就是多维前缀和．常见的多维前缀和的求解方法有两种．

#### 基于容斥原理

这种方法多用于二维前缀和的情形．给定大小为 𝑚 ×𝑛![](data:image/gif;base64,R0lGODlhAQABAIAAAAAAAP///yH5BAEAAAAALAAAAAABAAEAAAIBRAA7 "m\times n") 的二维数组 𝐴![](data:image/gif;base64,R0lGODlhAQABAIAAAAAAAP///yH5BAEAAAAALAAAAAABAAEAAAIBRAA7 "A")，要求出其前缀和 𝑆![](data:image/gif;base64,R0lGODlhAQABAIAAAAAAAP///yH5BAEAAAAALAAAAAABAAEAAAIBRAA7 "S")．那么，𝑆![](data:image/gif;base64,R0lGODlhAQABAIAAAAAAAP///yH5BAEAAAAALAAAAAABAAEAAAIBRAA7 "S") 同样是大小为 𝑚 ×𝑛![](data:image/gif;base64,R0lGODlhAQABAIAAAAAAAP///yH5BAEAAAAALAAAAAABAAEAAAIBRAA7 "m\times n") 的二维数组，且

𝑆𝑖,𝑗=∑𝑖′≤𝑖∑𝑗′≤𝑗𝐴𝑖′,𝑗′.![](data:image/gif;base64,R0lGODlhAQABAIAAAAAAAP///yH5BAEAAAAALAAAAAABAAEAAAIBRAA7 "S_{i,j} = \sum_{i'\le i}\sum_{j'\le j}A_{i',j'}.")

类比一维的情形，𝑆𝑖,𝑗![](data:image/gif;base64,R0lGODlhAQABAIAAAAAAAP///yH5BAEAAAAALAAAAAABAAEAAAIBRAA7 "S_{i,j}") 应该可以基于 𝑆𝑖−1,𝑗![](data:image/gif;base64,R0lGODlhAQABAIAAAAAAAP///yH5BAEAAAAALAAAAAABAAEAAAIBRAA7 "S_{i-1,j}") 或 𝑆𝑖,𝑗−1![](data:image/gif;base64,R0lGODlhAQABAIAAAAAAAP///yH5BAEAAAAALAAAAAABAAEAAAIBRAA7 "S_{i,j-1}") 计算，从而避免重复计算前面若干项的和．但是，如果直接将 𝑆𝑖−1,𝑗![](data:image/gif;base64,R0lGODlhAQABAIAAAAAAAP///yH5BAEAAAAALAAAAAABAAEAAAIBRAA7 "S_{i-1,j}") 和 𝑆𝑖,𝑗−1![](data:image/gif;base64,R0lGODlhAQABAIAAAAAAAP///yH5BAEAAAAALAAAAAABAAEAAAIBRAA7 "S_{i,j-1}") 相加，再加上 𝐴𝑖,𝑗![](data:image/gif;base64,R0lGODlhAQABAIAAAAAAAP///yH5BAEAAAAALAAAAAABAAEAAAIBRAA7 "A_{i,j}")，会导致重复计算 𝑆𝑖−1,𝑗−1![](data:image/gif;base64,R0lGODlhAQABAIAAAAAAAP///yH5BAEAAAAALAAAAAABAAEAAAIBRAA7 "S_{i-1,j-1}") 这一重叠部分的前缀和，所以还需要再将这部分减掉．这就是 [容斥原理](https://oi-wiki.org/math/combinatorics/inclusion-exclusion-principle/)．由此得到如下递推关系：

𝑆𝑖,𝑗=𝐴𝑖,𝑗+𝑆𝑖−1,𝑗+𝑆𝑖,𝑗−1−𝑆𝑖−1,𝑗−1.

实现时，直接遍历 (𝑖,𝑗)![](data:image/gif;base64,R0lGODlhAQABAIAAAAAAAP///yH5BAEAAAAALAAAAAABAAEAAAIBRAA7 "(i,j)") 求和即可．

> 考虑一个具体的例子．

![二维前缀和示例](https://oi-wiki.org/basic/images/prefix-sum-2d.svg)

其中，𝑆![](data:image/gif;base64,R0lGODlhAQABAIAAAAAAAP///yH5BAEAAAAALAAAAAABAAEAAAIBRAA7 "S") 是矩阵 𝐴![](data:image/gif;base64,R0lGODlhAQABAIAAAAAAAP///yH5BAEAAAAALAAAAAABAAEAAAIBRAA7 "A") 的前缀和．根据定义，𝑆3,3![](data:image/gif;base64,R0lGODlhAQABAIAAAAAAAP///yH5BAEAAAAALAAAAAABAAEAAAIBRAA7 "S_{3,3}") 是左图中虚线方框中的子矩阵的和．而且，𝑆3,2![](data:image/gif;base64,R0lGODlhAQABAIAAAAAAAP///yH5BAEAAAAALAAAAAABAAEAAAIBRAA7 "S_{3,2}") 是蓝色子矩阵的和，𝑆2,3![](data:image/gif;base64,R0lGODlhAQABAIAAAAAAAP///yH5BAEAAAAALAAAAAABAAEAAAIBRAA7 "S_{2,3}") 是红色子矩阵的和，它们重叠部分的和是 𝑆2,2![](data:image/gif;base64,R0lGODlhAQABAIAAAAAAAP///yH5BAEAAAAALAAAAAABAAEAAAIBRAA7 "S_{2,2}")．由此可见，如果直接相加 𝑆3,2![1](data:image/gif;base64,R0lGODlhAQABAIAAAAAAAP///yH5BAEAAAAALAAAAAABAAEAAAIBRAA7 "S_{3,2}") 和 𝑆2,3![1](data:image/gif;base64,R0lGODlhAQABAIAAAAAAAP///yH5BAEAAAAALAAAAAABAAEAAAIBRAA7 "S_{2,3}")，会重复计算 𝑆2,2![](data:image/gif;base64,R0lGODlhAQABAIAAAAAAAP///yH5BAEAAAAALAAAAAABAAEAAAIBRAA7 "S_{2,2}")，所以应该有

𝑆3,3=𝐴3,3+𝑆2,3+𝑆3,2−𝑆2,2=5+18+15−9=29.

同样的道理，在已经预处理出二维前缀和后，要查询左上角为 (𝑖1,𝑗1)![](data:image/gif;base64,R0lGODlhAQABAIAAAAAAAP///yH5BAEAAAAALAAAAAABAAEAAAIBRAA7 "(i_1,j_1)")、右下角为 (𝑖2,𝑗2)![](data:image/gif;base64,R0lGODlhAQABAIAAAAAAAP///yH5BAEAAAAALAAAAAABAAEAAAIBRAA7 "(i_2,j_2)") 的子矩阵的和，可以计算

𝑆𝑖2,𝑗2−𝑆𝑖1−1,𝑗2−𝑆𝑖2,𝑗1−1+𝑆𝑖1−1,𝑗1−1.![](data:image/gif;base64,R0lGODlhAQABAIAAAAAAAP///yH5BAEAAAAALAAAAAABAAEAAAIBRAA7 "S_{i_2,j_2} - S_{i_1-1,j_2} - S_{i_2,j_1-1} + S_{i_1-1,j_1-1}.")

这可以在 𝑂(1)![](data:image/gif;base64,R0lGODlhAQABAIAAAAAAAP///yH5BAEAAAAALAAAAAABAAEAAAIBRAA7 "O(1)") 时间内完成．

在二维的情形，以上算法的时间复杂度可以简单认为是 𝑂(𝑚𝑛)![](data:image/gif;base64,R0lGODlhAQABAIAAAAAAAP///yH5BAEAAAAALAAAAAABAAEAAAIBRAA7 "O(mn)")，即与给定数组的大小成线性关系．但是，当维度 𝑘![](data:image/gif;base64,R0lGODlhAQABAIAAAAAAAP///yH5BAEAAAAALAAAAAABAAEAAAIBRAA7 "k") 增大时，由于容斥原理涉及的项数以指数级的速度增长，时间复杂度会成为 𝑂(2𝑘𝑁)![](data:image/gif;base64,R0lGODlhAQABAIAAAAAAAP///yH5BAEAAAAALAAAAAABAAEAAAIBRAA7 "O(2^kN)")，其中 𝑘![](data:image/gif;base64,R0lGODlhAQABAIAAAAAAAP///yH5BAEAAAAALAAAAAABAAEAAAIBRAA7 "k") 是数组维度，而 𝑁![](data:image/gif;base64,R0lGODlhAQABAIAAAAAAAP///yH5BAEAAAAALAAAAAABAAEAAAIBRAA7 "N") 是给定数组大小．因此，该算法不再适用．


# 差分

差分是一种与前缀和相对的策略，是前缀和的逆运算．相较于给定某一序列求它的差分，竞赛中更为常见的情景是，通过维护差分序列的信息，实现多次区间修改．在区间修改结束后，可以通过前缀和恢复原序列的信息，实现对原序列的查询．注意修改操作一定要在查询操作之前．

如果需要支持多次修改和查询的混合操作，需要使用 [树状数组](https://oi-wiki.org/ds/fenwick/)，但是它们的思想是共通的．

### 一维差分

对于序列 {𝑎𝑖}![](data:image/gif;base64,R0lGODlhAQABAIAAAAAAAP///yH5BAEAAAAALAAAAAABAAEAAAIBRAA7 "\{a_i\}")，它的差分序列 {𝐷𝑖}![](data:image/gif;base64,R0lGODlhAQABAIAAAAAAAP///yH5BAEAAAAALAAAAAABAAEAAAIBRAA7 "\{D_i\}") 是指

𝐷𝑖=𝑎𝑖−𝑎𝑖−1, 𝑎0=0.![](data:image/gif;base64,R0lGODlhAQABAIAAAAAAAP///yH5BAEAAAAALAAAAAABAAEAAAIBRAA7 "D_i = a_i - a_{i-1},~ a_0 = 0.")

C++ 标准库中实现了差分函数 [`std::adjacent_difference`](https://zh.cppreference.com/w/cpp/algorithm/adjacent_difference)，定义于头文件 `<numeric>` 中．

前缀和与差分的关系如下：

设 {𝐷𝑖}![](data:image/gif;base64,R0lGODlhAQABAIAAAAAAAP///yH5BAEAAAAALAAAAAABAAEAAAIBRAA7 "\{D_i\}") 是 {𝑎𝑖}![](data:image/gif;base64,R0lGODlhAQABAIAAAAAAAP///yH5BAEAAAAALAAAAAABAAEAAAIBRAA7 "\{a_i\}") 的差分序列，那么，有

- 序列 {𝑎𝑖}![](data:image/gif;base64,R0lGODlhAQABAIAAAAAAAP///yH5BAEAAAAALAAAAAABAAEAAAIBRAA7 "\{a_i\}") 是序列 {𝐷𝑖}![](data:image/gif;base64,R0lGODlhAQABAIAAAAAAAP///yH5BAEAAAAALAAAAAABAAEAAAIBRAA7 "\{D_i\}") 的前缀和，即
    
    𝑎𝑖=𝑖∑𝑗=1𝐷𝑗.![](data:image/gif;base64,R0lGODlhAQABAIAAAAAAAP///yH5BAEAAAAALAAAAAABAAEAAAIBRAA7 "a_i = \sum_{j=1}^i D_j.")
- 序列 {𝑎𝑖}![](data:image/gif;base64,R0lGODlhAQABAIAAAAAAAP///yH5BAEAAAAALAAAAAABAAEAAAIBRAA7 "\{a_i\}") 的前缀和为
    
    𝑆𝑖=𝑖∑𝑗=1𝑗∑𝑘=1𝐷𝑘=𝑖∑𝑗=1(𝑖−𝑗+1)𝐷𝑗.
